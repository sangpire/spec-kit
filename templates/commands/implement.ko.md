## 사용자 입력

```text
$ARGUMENTS
```

진행하기 전에 사용자 입력을 **반드시** 고려해야 합니다 (비어있지 않은 경우).

## 개요

1. 저장소 루트에서 `{SCRIPT}` 실행 및 FEATURE_DIR 및 AVAILABLE_DOCS 목록 파싱. 모든 경로는 절대 경로여야 합니다. 'I'm Groot'와 같은 인수의 단일 따옴표는 이스케이프 구문 사용: 예: 'I'\''m Groot' (또는 가능하면 큰따옴표: "I'm Groot").

2. **체크리스트 상태 확인** (FEATURE_DIR/checklists/가 있는 경우):
   - checklists/ 디렉토리의 모든 체크리스트 파일 스캔
   - 각 체크리스트에 대해 계산:
     - 총 항목: `- [ ]` 또는 `- [X]` 또는 `- [x]`와 일치하는 모든 줄
     - 완료된 항목: `- [X]` 또는 `- [x]`와 일치하는 줄
     - 미완료 항목: `- [ ]`와 일치하는 줄
   - 상태 테이블 생성

   - **체크리스트가 불완전한 경우**:
     - 미완료 항목 수가 있는 테이블 표시
     - **중지** 및 질문: "일부 체크리스트가 불완전합니다. 구현을 계속 진행하시겠습니까? (예/아니오)"
     - 계속하기 전에 사용자 응답 대기
     - 사용자가 "아니오" 또는 "대기" 또는 "중지"라고 하면 실행 중단
     - 사용자가 "예" 또는 "진행" 또는 "계속"이라고 하면 3단계로 진행

   - **모든 체크리스트가 완료된 경우**:
     - 모든 체크리스트가 통과되었음을 보여주는 테이블 표시
     - 자동으로 3단계로 진행

3. 구현 컨텍스트 로드 및 분석:
   - **필수**: 완전한 작업 목록 및 실행 계획을 위한 tasks.md 읽기
   - **필수**: 기술 스택, 아키텍처 및 파일 구조를 위한 plan.md 읽기
   - **있는 경우**: 엔티티 및 관계를 위한 data-model.md 읽기
   - **있는 경우**: API 사양 및 테스트 요구사항을 위한 contracts/ 읽기
   - **있는 경우**: 기술적 결정 및 제약사항을 위한 research.md 읽기
   - **있는 경우**: 통합 시나리오를 위한 quickstart.md 읽기

4. **프로젝트 설정 확인**:
   - **필수**: 실제 프로젝트 설정을 기반으로 무시 파일 생성/확인
   - git 저장소인지 확인하여 .gitignore 생성/확인
   - Dockerfile* 존재 또는 plan.md에 Docker가 있으면 .dockerignore 생성/확인
   - 기술 스택에 따라 적절한 무시 파일 생성

5. **단계별 구현**:
   - Phase 1: 설정 작업 실행
   - Phase 2: 기초 작업 실행 (모든 사용자 스토리 차단)
   - Phase 3+: 우선순위 순서로 사용자 스토리 구현
   - 각 작업 완료 후 tasks.md에서 체크 표시
   - 각 단계 후 체크포인트 검증

6. **보고**: 각 단계 완료 후 진행 상황 업데이트, 완료 시 최종 요약 제공
